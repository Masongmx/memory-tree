# 🌳 Memory Tree — 让你的龙虾学会遗忘

> *"不是记忆太少，而是记住了太多不该记的。"*

---

## 你的龙虾是不是也这样？

用 OpenClaw 一段时间后，你大概经历过这些：

**"我都说了八百遍了！"**

你第八次告诉它"抓推文用 x-tweet-fetcher，别用浏览器"，它第八次说"网络有问题我看不到"。你手动在记忆文件里加了一条规则，管了两天，又忘了。

**"怎么越来越蠢了？"**

刚装好的时候它反应很快、回答精准。用了一两个月，它开始变慢、变模糊、经常答非所问。打开记忆文件一看——好家伙，塞了 3000 多字，技能清单、决策记录、API 密钥、聊天日志……什么都往里放。

**"清理完又变回原样"**

你终于忍不住手动精简了一次记忆文件。效果立竿见影。但三天后它又把新的一堆东西塞进去，一周后又回到臃肿状态。陷入循环。

**为什么？**

因为你的龙虾**不会遗忘**。

它记住了一切——重要的、不重要的、过时的、临时的，全部混在一起。就像一个人试图记住生命中的每一秒钟，结果就是什么都记不清。

人类的记忆之所以好用，恰恰是因为**我们会遗忘**。不重要的事情自然淡忘，反复回忆的事情刻骨铭心。遗忘不是 bug，是 feature。

Memory Tree 就是给龙虾装上这个 feature。

---

## 一棵什么样的树？

想象一棵记忆树：

- **🌱 新知识**像种子一样发芽
- **🌿 常用的知识**枝繁叶茂
- **不常用的知识**慢慢变黄
- **过时的知识**自然凋落
- **凋落的知识**化作土壤，滋养新的成长

关键在于——**这一切是自动发生的**。

你不需要告诉它"这条知识该删了"，也不需要手动整理。树有自己的生命节律：被反复翻阅的知识保持翠绿，长期不被触碰的知识自然褪色。

---

## 怎么做到的？

每条知识都有一个"新鲜度"分数（我们叫它**置信度**），满分 1.0。

**知识会变绿：**
- 被搜索到 → 分数涨一点点
- 被实际使用 → 分数涨更多
- 你手动标记为重要 → 直接拉满

**知识会变黄：**
- 核心原则（P0）→ 永不褪色
- 重要知识（P1）→ 每天没被翻到，轻微褪色，约 5 个月才会枯萎
- 日常笔记（P2）→ 每天没被翻到，明显褪色，约 3 个多月枯萎

枯萎的知识不会丢失——它的精华会被提取保存，就像落叶归根化作养分。但它的"肉身"会从记忆文件中移除，不再占用空间。

**最妙的地方：越常被搜到的知识越不容易褪色，越不容易褪色就越容易被搜到——这是一个良性循环。**

而这个搜索不是简单的关键词匹配，而是**语义搜索**——它能理解"X推文抓不到"和"抓推特内容用什么工具"说的是同一件事。这靠的是本地运行的 AI 模型，不花钱、不走网络、不泄露隐私。

## 效果怎么样？

这是我们自己的真实数据。

**治理前的记忆文件：** 超过 2000 字，什么都有——身份信息、技能清单、决策记录、待办事项、经验教训、情报指令、甚至还有明文 API 密钥。每次启动都要加载这一大坨东西，真正重要的信息反而被淹没了。

**治理后：** 约 600 字，只保留了真正跨会话有效的蒸馏内容。启动加载量减少 70%。

而且说实话，大多数人的记忆文件只会比我们更夸张。因为大部分人根本没有"记忆需要管理"这个意识——就像大多数人不会清理手机相册一样，直到 128GB 塞满才开始头疼。

Memory Tree 就是那个自动帮你整理相册的工具。不用你操心，它在后台默默工作。

---

## 用起来麻烦吗？

**不麻烦。**

安装完之后，你什么都不用做。

它会在后台自动运行：
- 定期扫描你的记忆文件，把新知识登记上
- 每天凌晨自动检查哪些知识该褪色了
- 太旧的知识自动归档，精华自动保留

当你想查看记忆树的状态时，它会给你一张清晰的健康报告：

```
🌳 记忆树
├── 📊 健康度: 71%
├── 🍃 总计: 7 条知识
│   ├── 🌿 绿叶: 5（活跃）
│   ├── 🍂 黄叶: 2（衰减中）
│   ├── 🍁 枯叶: 0
│   └── 🪨 土壤: 0（已归档）
```

绿得多 = 记忆健康。黄得多 = 有些知识该更新了。

## 和其他方案有什么不同？

OpenClaw 社区里已经有一些记忆管理方案了，比如 @loryoncloud 的 **Memory-Like-A-Tree**，给了我们很多启发。

我们的版本在他思路上做了几个不同选择：

1. **加了语义搜索** — 不只是知道"知识在不在"，还能理解"你在找什么"，即使你用的词和原文不一样也能找到。比如搜"抓推特"，能找到标题写着"X/Twitter 链接处理规则"的知识。

2. **用本地模型做搜索** — 语义搜索需要 AI 理解文本，通常要调用云端 API。我们直接用你电脑上已有的 Ollama 运行本地模型，零成本、零延迟、数据不出本机。

3. **作为 OpenClaw 技能发布** — 安装一次就集成进你的龙虾，不需要额外的目录结构或配置文件，兼容你现有的记忆系统。

## 几个你可能关心的问题

**Q：会删除我的记忆吗？**
A：不会。枯萎的知识会被"归档"，精华被提取保留，不是真的删除。你可以随时查看归档内容。

**Q：需要额外安装什么？**
A：只需要 Ollama（大多数 OpenClaw 用户已经有了）和一个 embedding 模型。安装命令就一行：`ollama pull qwen3-embedding`。

**Q：会影响龙虾的运行速度吗？**
A：不会，反而会更快。记忆文件更精简 = 每次启动加载更少内容 = 省钱省时间。

**Q：它会消耗 API 额度吗？**
A：完全不会。所有操作都在你本地完成，不调用任何云端 API。

**Q：我已经有很多历史记忆了，能用吗？**
A：能用。第一次运行时它会自动扫描你现有的记忆文件，全部登记建档，不需要手动迁移。

## 最后说两句

很多人在给龙虾搭各种花哨的架构——三层记忆、向量数据库、知识图谱。搭完之后发现，龙虾根本不会主动用这些架构，文件夹空空如也。

Memory Tree 的思路是反过来的：**不要搭空壳，先让现有的东西活起来。**

你的龙虾每次启动都会读 MEMORY.md。不需要引入新的存储后端、新的概念框架。只需要给它一个属性——**置信度**。然后让置信度自动变化。

被用的变绿，不被用的变黄，枯萎的归档。

这就是一棵树。它会自己生长。
