# 🌳 记忆树

> OpenClaw 的记忆管理技能。让 agent 记住该记的，忘记该忘的。

记忆树解决三个问题：

1. **搜得到** — 语义搜索，找到你想要的记忆
2. **记得住** — 自动提取 + 手动标记，记住重要的事情
3. **忘得掉** — 过期清理，避免记忆无限膨胀，且不会误删

## 快速开始

```bash
# 搜索记忆（你说："你记不记得 xxx"）
python3 scripts/memory_tree.py vector-search "xxx"

# 标记重要记忆（你说："记住了"）
python3 scripts/memory_tree.py mark "标题"

# 查看记忆统计
python3 scripts/memory_tree.py stats
```

## 功能

| 功能 | 命令 | 说明 |
|------|------|------|
| 向量搜索 | `vector-search "查询"` | 语义搜索，不再只是关键词匹配 |
| **重要性评分** | `importance "标题"` | **v3.0 新增**：自动评估记忆保留价值 |
| 标记永久 | `mark "标题"` | 📌 标记永不衰减 |
| 冲突检测 | `conflict` | 检测可能冲突的记忆 |
| 遗忘预测 | `decay` | 提醒你哪些记忆可能不再有用 |
| 自动提取 | `auto-extract --limit 5` | 从对话历史自动提取 |
| 健康评分 | `health` | 四维度评分 |
| 同步 | `sync` | MEMORY.md → 向量库 |
| 统计 | `stats` | 查看记忆条数和分类 |
| 周报 | `weekly` | 本周统计 |

## 用户怎么说，Agent 怎么做

| 用户说的话 | Agent 的动作 |
|------------|-------------|
| "记住了" / "这个很重要" | 标记为永久记忆，检查是否与旧记忆冲突 |
| "你记不记得 xxx" | 语义搜索相关记忆 |
| "你怎么又忘了" | 同步 MEMORY.md 到向量库 |
| "记忆统计" | 显示记忆条数和分类 |
| "生成周报" | 生成本周统计 |

**Agent 主动触发**：
- 每周定期：遗忘预测 + 健康评分 + 自动提取
- 标记记忆时：自动检查冲突

## 技术栈

- **Embedding**: qwen3-embedding（本地，4096维）
- **LLM**: qwen3:8b（本地）
- **存储**: SQLite + 向量
- **成本**: 零（全本地运行）

## 记忆分类

| 类型 | 标记 | 说明 |
|------|------|------|
| 永久记忆 | 📌 | 永不衰减，核心规则 |
| 重要记忆 | [P0] | 缓慢衰减，定期复习 |
| 一般记忆 | — | 正常衰减，30天清理 |
| 临时记忆 | — | 快速衰减，7天清理 |

## 遗忘规则

**什么该遗忘**：
- ✅ 已完成的任务
- ✅ 过期的待办
- ✅ 临时信息

**什么不该遗忘**：
- ❌ 核心偏好
- ❌ 永久规则
- ❌ 📌 标记的记忆

**遗忘预测不会自动清理**，只会提醒你确认。

## 冲突检测

当检测到可能冲突的记忆时，Agent 会提醒你确认：

```
⚠️ 发现冲突："老板不喜欢爹味说教" vs "文章要有深度分析"
💬 这种情况下应该怎么做？
```

Agent 不会自己判断，会等你确认。
